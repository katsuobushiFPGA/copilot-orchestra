# AGENTS.md - Copilot CLI ワークフロー定義

## プロジェクトワークフロー

このプロジェクトでは、以下のワークフローに従ってタスクを管理・実行する。

### ワークフロー概要

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────┐
│ タスク分割 │───→│ 並列開発  │───→│ AIレビュー │───→│ 人間レビュー│───→│ 完了  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────┘
                     ↑               │                │
                     └───────────────┘                │
                     (差し戻し)        └────────────────┘
```

### 使用エージェント

| エージェント | 役割 | 使い方 |
|------------|------|--------|
| `orchestrator` | タスク分割・フロー管理 | メインのやり取りで使用 |
| `developer` | コード実装（TDD） | サブエージェントとして並列実行 |
| `reviewer` | AIコードレビュー | タスク完了後に自動実行 |
| `explainer` | 人間向け解説 | AIレビュー後、人間レビュー前に自動実行 |

### 開始方法

1. **タスクを定義する**: orchestrator エージェントに要件を伝える
2. **自動分割**: orchestrator がタスクを分割し `TASKS.md` に記録する
3. **並列実行**: `/fleet` モードで developer エージェントが TDD で並列に実装する
4. **AIレビュー**: 完了後、reviewer がコードレビューする
5. **解説**: explainer が実装内容をわかりやすく解説する
6. **人間レビュー**: 解説を読んだ上で承認 or 差し戻し
7. **完了**: すべてのレビューを通過したタスクが `done` になる

### コマンド例

```
# orchestrator エージェントでワークフローを開始
Use the orchestrator agent to break down and manage this task: [要件の説明]

# fleet モードを有効にして並列開発
/fleet

# タスクの状態を確認
cat TASKS.md
```

### 通知設定

通知は以下の方法で送信される:
- **ターミナル出力**: 常に有効
- **デスクトップ通知**: `notify-send` が利用可能な場合
- **Webhook**: 環境変数 `NOTIFY_WEBHOOK_URL` を設定した場合（Slack等）

```bash
# Slack Webhook の例
export NOTIFY_WEBHOOK_URL="https://hooks.slack.com/services/YOUR/WEBHOOK/URL"
```

### タスク管理ルール

- すべてのタスクは `TASKS.md` で追跡する
- タスクは必ず「AIレビュー → 人間レビュー」の両方を通過すること
- 人間レビューは省略できない
- 差し戻しは何度でも可能
